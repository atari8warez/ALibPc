Introduction

ALibPc is a companion utility to AspeQt. While AspeQt is a PC based storage solution, providing virtual disk drives to an Atari computer using an SIO2PC interface, ALibPc allows you to copy your real Atari 8-bit floppy disks to your PC, using your Atari disk drive connected via a 10502PC interface. Floppy disks are converted to disk images in popular ATR file format. Those ATR files can later be served to your Atari computer using AspeQt or other similar Atari peripheral software. ALibPc will also allow you to file these disk images into an Atari Software database for archival and informational purposes. You can categorize the entries in the database using various software categories and can query or search the database using those categories as search criteria. You can also sort your data, save/restore your database etc. Note that ALibPC is not a copy protected disk copier, and it will only work with non-protected disks.

 

System Requirements

 

10502PC Hardware

First, you will need to have a 10502PC device to connect your Atari disk drive(s) to your PC. The device can either be a RS232 Standard COM Port or a USB Virtual COM Port based hardware. As far as ALibPc is concerned they are exactly the same thing, a serial port.

Usage at normal (19,200bp) SIO speeds

All PC COM ports (RS232 or VCP) offer this standard baud rate that all Atari disk drives use to communicate with the computer. Many Serial-to-USB converter devices (and thus 10502PC devices) use FTDI UARTs. Their port drivers come in two flavors. One is a direct driver (called D2xx), the other one is the VCP (aka. Virtual COM Port) serial port driver. ALibPc uses the VCP driver to maintain greatest possible compatibility with all available 10502PC devices at regular SIO speeds. Using the direct driver with API calls would have limited the device support to FTDI UART based products.

10502PC devices supported at regular speed are:

·         Atari8Warez SIO2PC/10502PC Dual-USB

·         Lotharek 10502PC/SIO2PC-USB

·         AtariMax Universal SIO2PC/ProSystem Interface (RS232 serial version only)

·         Any other 10502PC device based on either RS232 or VCP serial port technology

 

Usage at high SIO speeds

RS232 and VCP drivers normally only support standard serial port baud rates, but most speed enhanced Atari floppy drives use non-standard rates for high-speed operation. This requires custom serial port drivers that can communicate at those speeds. RS232 drivers are included with Windows, and are not customizable. So that limits the high-speed operation to devices that use a Virtual COM Port.  Since FTDI is the most popular brand, and they allow driver customization, it is our natural choice for high-speed support.

VCP based high-speed operation comes with a price though. There are some inconveniences for the user:

·         Customized drivers mean 64bit versions of Windows must be booted with “Driver Signature Enforcement” disabled, because customization invalidates original driver signatures. You will need to boot your Windows version this way every time you need to install an unsigned driver, or change your windows configuration to boot in “Test Mode”.

 

·         Windows must be setup not to download device drivers from Windows Update so that your custom drivers are not overridden by regular FTDI drivers.

 

Please consult the “ALibPc Installation Guide” for more details on the above.

 

10502PC devices supported at high speed are:

·         Atari8Warez SIO2PC/10502PC Dual-USB

·         Lotharek 10502PC/SIO2PC-USB

 

PC Hardware

Then you will need a PC. Most PC's with minimal hardware configuration and a USB 2.0 port (or an RS232 COM port) will work. However high-speed operation may be adversely affected if the PC has an older and slower CPU, high CPU load, high disk activity, or low RAM.

 

PC Operating System

Windows OS (WinXP and above), no other version of Windows is supported. ALibPc is a 32bit application and will run on both 32 and 64bit versions of Windows. Linux (Ubuntu) version may be released after the development is completed on the Windows platform. Mac OS will not be supported.

 

AspeQt-A8W Edition is not required but highly recommended for full integration of the two applications. Other versions/forks of AspeQt are not supported. Make sure you have the latest version downloaded from here.

 

Known issues and technical limitations

 

ALibPc is developed using Atari8Warez “SIO2PC/10502PC Dual-USB”, and Lotharek 10502PC/SIO2PC-USB interfaces, and a “MegaSpeedy” enhanced Atari 1050 drive. MegaSpeedy emulates a handful of different Atari 1050 drive enhancements like HAPPY, SPEEDY, USDoubler, 1050 Turbo etc... ALibPc has been tested with all these emulated drives and will work with most of them (and with many others) in normal speed. High Speed is currently only supported on HAPPY, SPEEDY, USDoubler and any other drive which is compatible with those 3 drive types. I don't own any drives other than this upgraded 1050, so it is impossible for me to test the application with drives like XF551, INDUS GT etc. Expect only normal speed operation at best with those drive types until and if I ever get my hands on one to develop and test. (Contact me at ray-dot-ataergin-at-gmail-dot-com if you are willing to donate/loan one of these disk drives so that I can add high-speed support for them)

 

Here’s a summary of what is supported with various Atari compatible drive enhancements:

 

As of version 1.0 ALibPc can recognize and make use of the following Atari drives:

 

·         Stock 810/1050 (supported with normal speed).

·         USDoubler enhanced 1050 (supported with US sector skew formatting (1) and high speed).

·         Speedy enhanced 1050 (supported with track buffering and high speed)

·         Happy enhanced 810 (supported with track buffering) (2).

·         Happy enhanced 1050 (supported with track buffering, and high speed) (3).

·         All drive types emulated by the MegaSpeedy Upgrade (1050 Duplicator, Super Archiver and SuperMax are recognized as USDoubler Compatibles and supported with US sector skew formatting (1) and high speed).

·         1050 Turbo not supported or even recognized as it requires clock signal from the computer to be present, and the 10502PC interfaces do not provide the required signal.

·         Any other Atari SIO compatible drive maybe supported as a stock Atari drive with normal SIO speed.

·         Double sided/quad density disks are not supported.

 

 

Remember ALibPc is not an emulator, it is a disk copying software and takes full advantage of the newer technology when copying disks. So please don't compare it to an emulator and expect an accurate emulation of the real hardware. SIO sound emulation is provided as a vintage touch, and it can be turned off if it becomes annoying.

 

 

(1) US sector skew is only used when formatting a disk and not when reading or writing sectors from a disk. This is due to the strict timing required to successfully implement interleaved sector access which is not possible with a USB based Virtual COM port device on the PC. The USB protocol buffers the data and only transmits when the buffer is full making the correct timing for individual sector access impossible. Due to this restriction, disks formatted with Ultra Speed sector skews will actually be read slower than standard sector skew formatted ones on the PC. However, when the disk is used with an Atari computer under SpartaDos, Ultra Speed sector skews will function as expected.

 

(2) Happy 810 high-speed operation is not currently implemented. Disk operations will be performed at normal SIO speed.

 

(3) ALibPc uses the USDoubler compatible high-speed protocol of the HAPPY 1050 drive for high-speed operation. This requires firmware Revision 2 on the HAPPY 1050. Earlier revisions will run on normal speed.

 

Making Disk Images From Your Floppy Disks

 

Single drive operation
ALibPc main application Window contains 4 disk slots, this means you can daisy-chain up to 4 Atari drives and connect them to your 10502Pc.
When started, ALibPc will scan the SIO bus and detect any drive that is turned ON. You can also scan for available drives any time after initial start-up by pressing shortcut key ALT+S, or by selecting the option from the File menu.  Slots for the detected drives will be enabled for operation.

Clicking on the “Make Image” icon  will start the ATR making process. If you haven’t already assigned a file name for the ATR image by clicking on the “Folder Name bar” identified by the  icon , (which also act as a progress bar when the operation is underway), you will be presented a dialog to select/enter a file name and asked to ready the disk to continue.

If “Use track buffers when available” option is checked in the “General” section of the Preferences menu, ALibPc will enable read buffering for drives that support it.

During operation, any informational/warning or critical messages will be displayed in the log window below the disk slots.

If “Ask to mount created ATR to AspeQt” option is checked in the “General” section of the Preferences menu, ALibPc will ask whether the ATR should be mounted to AspeQt for immediate testing/use. If you answer YES, ALibPc will invoke AspeQt and pass the ATR file name, and AspeQt will mount the disk to the first available drive slot

If “Ask to add created ATR to the Library” option is checked in the “General” section of the Preferences menu, ALibPc will ask whether it should add the disk information into its Atari software database.

You can answer YES or NO to these questions, and if you don’t want to see them asked every time you make an ATR, you can save your choices for future reference. ALibPc will use the file name portion of the file as the software “Title”, and the “Category”, “Sub-Category” and some other information fields will be defaulted from the settings in the “Preferences/Library” menu. The information can later be edited via “Library/Browse/Edit” menu option.

You can hide/show the log window of ALibPc by pressing CTRL+SHIFT+W key combination, or by selecting the option from the Window menu. You can also open a separate “searchable” log window, just press key combination CTRL+SHIFT+L, or select the option from the Window Menu.

ALibPc makes use of extensive “tooltips” on the user interface items/icons like   etc., you can simply move your cursor over those elements to see information/help about various aspects of the application.

Multi Drive Operation

ALibPc supports batching of disk copying (and other) tasks. For example, you can specify up to 4 ATR file names, one on each enabled drive slot, and let the application create ATRs from up to 4 disk drives, uninterrupted. Simply make sure that all file names are specified by clicking on the “Folder name bar” of each drive slot and selecting a file name, and by pressing key combination ALT+SHIFT+M, or by selecting the option from the File menu to make multiple ATRs.

Writing Disk Images to your Disk(s)

 

Single Drive Operation

Clicking on the “Write Image” icon  will start the disk making process. If you haven’t already chosen an ATR file by clicking on the “Folder Name bar” identified by the  icon  , you will be presented a dialog to select a file from your PC, and asked to ready the disk to continue. During operation, any informational/warning or critical messages will be displayed in the log window below the disk slots.

If you have checked the “Format disk before write” option in the Disk Write Options section of the Preferences menu, ALibPC will first format the disk. If the drive is USDoubler (or compatible) enhanced, then ALibPC will, depending on the selection you made for the “Use Ultra Speed sector skew” option in the “Disk Write Options” section of the Preferences menu, use the special sector skew to make your disk high-speed capable under SpartaDos. The density of the disk for formatting will be determined from the ATR file.

After the format is completed, ALibPc will either use sector-by-sector, or track-by-track writes, depending on the selection you made for the “Use track buffers when available” option in the “General” section of the Preferences menu.

“Write empty sectors”, and “Verify writes” options in the “Disk Write Options” section of the Preferences menu will also be honored during disk writes. Generally speaking, there is no need to Verify Writes as the drives are normally reliable enough to write the sectors correctly, and checking this option slows the write operation down considerably.

 

Multi Drive Operation

You can specify up to 4 ATR file names, one on each enabled drive slot, and let the application write disks from up to 4 ATR images, uninterrupted. Simply make sure that all file names are specified by clicking on the “Folder name bar” of each drive slot and selecting a file name, and by pressing key combination ALT+SHIFT+W, or by selecting the option from the File menu to make multiple ATRs. 

 

Formatting Disk(s)

 

Single drive operation

 Clicking on the “Format Disk” icon  will start ALibPc format utility. Here you can select the density of the disk to be formatted from a drop-down list, and in the case of a USDoubler enhanced drive you can override the Ultra Speed sector skew option you previously set via the Preferences menu. The format utility gives you two different options to start formatting. If you click the “Start” button ALibPc will wait until the format finishes and will be able to return success/failure status for the format.  If you click “Submit” ALibPc will not wait after it issues the format command to the drive, and you will be able to return back to the main window to perform other tasks, but ALibPc won’t be able to return the status of the format operation.

 

Multi Drive Operation

You can also format multiple disks at once by selecting “Format multiple disks” option from the File menu or by pressing ALT+SHIFT+F key combination.  Here you can set disk densities and US sector skew options just like in the single drive operation. Clicking “Submit” will start formatting all available disks at the same time, or you can select to wait for each operation to finish by clicking “Start”.

Building an Atari 8 bit software Library

 

ALibPc not only allows you to copy your Atari 8 bit disks into disk images and vice versa, but also allows you to build a software library from the disks you copied to your PC. As you make your disk images, ALibPc inserts an entry for each image into its database using some default values, some of which you can define in the Library section of the Preferences dialog. The database system used in ALibPc Librarian is SQLite, a lightweight SQL based database system which does not require a server. This means you don’t have to worry about installing a database system on your PC for the Librarian functionality to work. ALibPc already contains the plug-ins required to run the database. As of version 0.5 ALibPc Librarian provides ways to edit, add, delete, sort, and filter library entries, plus some database maintenance options such as Backup, Restore and Move. Backing-up or Restoring the library is pretty obvious and requires little explanation. Suffice to say that the backup file is created in the same folder as the database file - ALibPc.db with the name ALibPc.db.bak. If you’d like to keep it somewhere else on your PC you’d have to move it manually using the facilities of Windows Explorer. The library file is initially created in ALibPc application folder. The “Path to the library” entry in the Library section of the Preferences dialog will be initially blank. A blank entry there means the database file resides in the same folder as the application. If you want to move it somewhere else on your PC, simply use the “Browse” (…) button to select the new location. The file will be moved to its new place and ALibPc will be restarted. The need for restart is due to how the database mechanism works behind the scenes. Note that if a backup file already exists, it won’t be moved along with the database. Future versions of ALibPc may implement that functionality. Also note that if you take a backup after moving the database to a new location, the new backup file will be created in the same new location as the database.

ALibPc Librarian is not a full-fledged, general purpose database management/editing tool. If you require such a tool to manipulate the database in different ways, I would recommend the “DB browser for SQLite” by Mauricio Piacentini. It may not be the best SQLite database browser/editor out there, but it certainly is a very nice product and it is free.

 

Setting Librarian Options

As it was already mentioned above, ALibPc adds a new entry to its library database every time you create a disk image from one of your disks. This is not automatically decided however. Since v0.5 ALibPc, after creating a disk image, will ask you whether you’d like to add information about this disk into your library. You can answer YES or NO and save your preference for future disks. If you don’t save it, ALibPc will ask you the question every time you create a new ATR. If you save your preference and change your mind later, you can make ALibPc to ask the question once again every time you create an ATR by checking “Ask to add created ATR to the library” option in the General section of the Preferences dialog.

 

Figure 4 - General Preferences

 

 

You can also define default values for Software Category, Sub Category, Language, Video Standard used, whether BASIC is required or not, and the system compatibility.  Category, Sub Category, and Language fields are editable, meaning you can add your own value if a suitable one is not found in the included list of values. You can only add one custom value, so if you change it later to another one the previous one will be lost from the list.
